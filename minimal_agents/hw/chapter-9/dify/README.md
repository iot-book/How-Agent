# 第九章作业：Dify 应用搭建

这一目录对应教程中的“第九章 低代码框架 Dify”。本章作业的目标，不是继续补 Python 代码，而是亲手搭建一个可运行的 Dify 应用，并记录自己的设计过程。

## 作业目标

请基于 Dify 完成一个实际可演示的应用。应用类型不限，可以是聊天助手、Chatflow、工作流或 Agent，只要能够体现你对节点编排、知识库接入或工具能力扩展的理解即可。

## 参考方向

- 课程资料问答助手
- RSS 订阅与内容总结助手
- 文档格式化与结构化输出助手
- 基于知识库的专题问答应用

## 提交内容

建议至少包含以下内容：

1. 你要解决的问题是什么；
2. 你选择了哪一种 Dify 应用形态；
3. 工作流或应用的核心节点设计；
4. 是否接入了知识库、工具或插件；
5. 最终运行效果截图与说明；
6. 你在搭建过程中遇到的问题与思考。

## 补充说明

本章作业更偏向体验与设计思考，重点是理解：

- 低代码平台怎样组织智能体应用；
- 哪些能力可以不用手写代码完成；
- 当工作流变复杂后，如何保持结构清晰。
